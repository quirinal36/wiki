# 🚀 셀피쉬클럽(AAA팀) 운영 모델 복제 가이드

> **원본 출처:** [빌더조쉬 × 젬마(신주혜)님 인터뷰](https://maily.so/josh/posts/e9o0y2mnr8w)
> **핵심 시스템:** 옵시디언 + 깃허브 + 클로드 코드 3단 구조로 비개발자 팀이 AI 에이전트 워크플로우를 구축한 사례

---

## 📌 1단계: 팀 구성 (1주차)

### 👥 인원 구성
- **6~8명**이 적정 규모 (AAA팀은 8명)
- **비개발자 중심**이 핵심 — 개발 지식 없어도 시작 가능
- 다양한 배경의 사람들이 모일수록 **워크플로우 조합의 다양성**이 커짐

### 🎯 핵심 마인드셋: "이기적 공유"
```
내가 배운 것, 삽질한 것, 성공한 것을 공유하는 것이
결국 나에게도 가장 큰 도움이 된다는 신념
```
- 결과물보다 **워크플로우(과정)** 가 더 가치 있다
- "비빔밥이 나왔어"가 아니라 "재료를 어떻게 공수하고 조리했는지"를 공유

---

## 📌 2단계: 기술 환경 구축 (1~2주차)

### 🧰 필요한 도구 3종 세트

| 도구 | 역할 | 비용 |
|------|------|------|
| **Obsidian** | 작업 공간 — 마크다운 작성 | 무료 |
| **GitHub** | 클라우드 저장소 + 배포 파이프라인 | 무료 |
| **Claude Code** | AI 두뇌 — 분석, 자동화, 콘텐츠 생성 | 유료 (Pro $20/월 추천) |

### 1. 옵시디언(Obsidian) 설치 및 설정
```bash
# 다운로드: https://obsidian.md/
# 각 팀원이 설치 후, 공통 Vault 폴더 구조 설정
```

#### AAA팀의 폴더 구조 (그대로 가져가도 충분)
```
vault/
├── missions/          # 주차별 미션 제출 (팀원이 직접 작성)
│   ├── week-01/
│   ├── week-02/
│   └── ...
├── 01-meetings/       # 위클리 회의록 (처음엔 노션 MCP로 가져와도 됨)
├── skills-site/       # 유용한 AI 스킬/플러그인 정보 공유
├── sharing/           # 미션 수행 후 공유 초안 (클로드와 협업)
├── analysis/          # ⭐ 자동화 핵심 — 클로드 코드가 분석 리포트 적재
│   ├── members/       #   사람별 참여 분석
│   └── weekly/        #   주간 흐름 분석
├── linkedin/          # 링크드인 글 자동 생성 결과물
└── vault-log/         # 폴더 구조 변경 이력 (충돌 방지)
```

### 2. GitHub 설정
```bash
# 2개의 레포지토리가 필요

# (1) 옵시디언 원본 레포 — 내부용, 팀원 작성물 보관
git init
git remote add origin git@github.com:team/vault-origin.git

# (2) aaa-admin 레포 — 외부 공개용, Vercel 배포 연결
git init
git remote add origin git@github.com:team/aaa-admin.git
```

#### GitHub 워크숍 진행
> AAA팀은 **팀 단위 GitHub 워크숍**을 먼저 진행했습니다.
> 비개발자들도 `git add / commit / push / pull` 기본 조작을 익혀야 합니다.

### 3. Claude Code 설정
```bash
# Claude Code 설치
npm install -g @anthropic-ai/claude-code
claude login

# 프로젝트 폴더에서 시작
cd vault/
claude
```

## 📌 3단계: 주간 루틴 정착 (3주차~)

### 🔄 주간 사이클

```
[월~토] 개인 미션 수행 → Obsidian에 작성 → GitHub push
[일요일] 주간 회의 → 회의록 작성
[월요일 아침] 자동화 실행 → 분석 리포트 생성 → 사이트 배포
```

### 📝 미션 운영 방식

**매주 하나씩 미션을 내고, 팀원들이 수행 후 공유:**

예시 미션:
- 1주차: "Claude Code로 나만의 스킬 하나 만들어보기"
- 2주차: "AI 도구 2개 조합해서 결과물 만들어보기"
- 3주차: "링크드인 글 자동 생성 스킬 만들어보기"
- 4주차: "내 프로젝트에 AI 에이전트 적용해보기"

**공유 포맷:**
```markdown
# [주차] 미션 제목

## 시도한 것
- ...

## 막혔던 부분 (삽질 기록)
- ...

## 해결 방법
- ...

## 결과물 / 인사이트
- ...

## 사용한 도구 & 대화 링크
- Claude Code: [링크]
- ChatGPT: [링크]
```

### 📋 회의록 템플릿
```markdown
# YYYY-MM-DD 위클리 회의

## 각자 진행 상황
- [이름]: ...

## 이번 주 인사이트
- 포인트 1
- 포인트 2

## 다음 주 미션
- [공통 미션]

## 결정 사항
- ...
```

## 📌 4단계: 자동화 시스템 구축 (4~6주차)

### 🤖 Claude Code로 만들 자동화 스킬

#### (1) Publish 스킬 (콘텐츠 자동 배포)
```
기능: 지정된 폴더의 마크다운을 → HTML로 변환 → Vercel 자동 배포
트리거: 월요일 아침 cron 실행
대상: missions/, skills-site/ (공개 허용된 폴더만)
```

#### (2) 구간 분석 자동 생성
```
기능: 과제 + 회의록 전체를 읽어서 → 주간/개인별 분석 리포트 생성
출력: analysis/ 폴더에 적재
- analysis/members/김다솔.md → "이 사람의 관심 키워드, 참여 패턴, 성장 흐름"
- analysis/weekly/week-04.md → "이번 주 팀 전체의 흐름과 인사이트"
```

#### (3) 링크드인 글 자동 생성
```
기능: 미션 수행 내용을 기반으로 → 링크드인 게시글 초안 생성
출력: linkedin/ 폴더에 적재
```

#### (4) Vault Log (변경 이력 추적)
```
기능: 폴더 구조 큰 변경 시 → 스냅샷 + 변경 사유 기록
용도: 다수가 동시 편집할 때 충돌 방지
```

### ⏰ Cron 자동화 스케줄 (AAA팀 방식)
```
# 월요일 오전 9시 — Publish 실행 (사이트 배포)
0 9 * * 1 /path/to/publish-skill.sh

# 월요일 오전 8시 — 분석 리포트 생성
0 8 * * 1 /path/to/analysis-skill.sh

# 매일 자정 — Vault 변경 감지 및 로그
0 0 * * * /path/to/vault-log-skill.sh
```

## 📌 5단계: 외부 공개 & 커뮤니티 확장 (6주차~)

### 🌐 공개 구조
```
[내부] Obsidian → GitHub(private repo)
  ↓ Publish 스킬 (공개 폴더만 선별)
[외부] GitHub(public repo) → Vercel 자동 배포 → 웹사이트
```

### 📊 콘텐츠 선순환
```
회의록 작성 → 분석 리포트 생성 → 인사이트 도출 → 외부 공유 → 새 멤버 유입
      ↑                                                              ↓
      └─────────── 더 많은 데이터 = 더 풍부한 콘텐츠 자산 ←──────────┘
```

## 🎯 당장 이번 주에 할 일

### ☑️ 체크리스트

**1주차:**
- [ ] 팀원 모집 (6~8명, 다양한 배경)
- [ ] "이기적 공유" 개념 공유 및 합의
- [ ] Obsidian 설치 + 공통 Vault 폴더 구조 생성
- [ ] GitHub 레포 2개 생성 + 팀원 초대
- [ ] Claude Code 설치 및 기본 사용법 익히기

**2주차:**
- [ ] GitHub 워크숍 진행 (git 기본기)
- [ ] 첫 미션 출제 + 수행
- [ ] 첫 주간 회의 진행
- [ ] 회의록 Obsidian에 작성

**3~4주차:**
- [ ] Publish 스킬 개발 (Claude Code 활용)
- [ ] Cron 자동화 설정
- [ ] 외부 공개 사이트 연결 (Vercel + GitHub Pages)

**5~6주차:**
- [ ] 분석 자동화 스킬 개발
- [ ] 링크드인 글 자동 생성 스킬 개발
- [ ] 시스템 개선 + 안정화

## 💡 성공을 위한 핵심 원칙

1. **완벽함보다 실행** — "이 정도 근거면 도전해볼 만하다" 마인드
2. **워크플로우가 자산** — 결과물보다 과정을 기록하고 공유
3. **사람은 판단만** — 실행은 AI가, 의사결정은 사람이
4. **계속 개선** — "오늘 버전이 완성형은 아니다"라는 마음가짐
5. **함께 즐겁게** — "같이 즐겁게 하는 게 가장 오래 갈 수 있는 방식"
