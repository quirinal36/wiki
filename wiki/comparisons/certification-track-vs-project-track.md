# Certification Track vs Project Track in Coding Education

## What is being compared
The source contrasts two academy choices for a 13-year-old student:
- certification-first instruction aimed at passing a credential exam
- project-first instruction aimed at building web services with AI support

## Why the comparison matters
The article uses this choice to argue that AI has changed what good coding education should optimize for.

## Comparison table
| Dimension | Certification track | Project track |
| --- | --- | --- |
| Primary goal | Pass a credential exam | Build a real service |
| Main artifact | Test score / certificate | Working project / portfolio |
| AI role | Optional helper | Core builder's tool |
| Student posture | Follow the syllabus | Frame problems and iterate |
| Long-term relevance | Useful but limited | Better aligned with AI-era work |

## Verdict / synthesis
The article clearly favors the project track when the goal is future readiness, [[learner-agency]], and practical service creation. The certification track can still be useful as a supplemental path, but it should not define the entire program. The later C-language essay makes the same point in a syntax-first programming context, reinforcing the shift from drill-based instruction to AI-assisted project work.
The parent-facing essay adds one more layer: coding can still matter as thinking training, but that value should be paired with problem solving and self-direction rather than exam-only drill.
