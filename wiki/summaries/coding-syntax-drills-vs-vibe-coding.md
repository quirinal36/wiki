# Coding Syntax Drills vs Vibe Coding

## What this source is
A Korean essay arguing that syntax-first programming education is being overtaken by AI-assisted project creation and that students should focus on planning, directing AI, and solving problems.

## Key takeaways
- The article uses university C-language instruction as an example of outdated syntax-first education.
- It argues that AI has already changed coding enough that memorizing language details is no longer the main differentiator.
- "Vibe coding" is presented as the new default for many developers.
- Students should learn to specify what they want to build, what to change, and how to resolve blockers.

## Important claims
- The old equation "algorithm practice = coding skill" no longer holds in the same way.
- Students need project ownership, not just syntax recall.
- Good education should cultivate planning, creativity, and problem-solving.
- This source extends [[certification-track-vs-project-track]] and [[ai-service-building-curriculum]] into the programming-syntax domain.
