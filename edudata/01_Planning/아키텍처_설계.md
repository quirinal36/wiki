# 🏗️ 아키텍처 설계 및 기술 스택

## 1. 기술 스택 (Tech Stack)
류승엽, 김우진 학생과의 원활한 협업과 빠른 MVP 구현을 위해 아래 스택을 선정했습니다.

- **Frontend**: `Next.js 14+ (App Router)`
  - 이유: 상담 페이지(SSR)와 관리자 대시보드를 하나의 프레임워크로 구현 가능.
- **Styling**: `Tailwind CSS` + `shadcn/ui`
  - 이유: 일관된 디자인 시스템을 빠르게 구축.
- **Backend/Database**: `Supabase`
  - 이유: 실시간 상담 채팅 저장 및 사용자 인증(Auth) 해결.
- **AI Integration**: `Vercel AI SDK`
  - 이유: LLM 스트리밍 답변 구현 및 도구 호출(Tool Calling) 최적화.
- **Language**: `TypeScript`
  - 이유: 타입 정의를 통해 협업 시 발생하는 런타임 에러 방지.

## 2. 데이터 흐름 (Data Flow)
1. **상담 입력**: 학생이 채팅 창에 고민 입력.
2. **AI 처리**: Claude 3.5 Sonnet 모델이 상담 페르소나를 입고 답변 생성.
3. **데이터 저장**: 모든 대화 내역은 Supabase의 `messages` 테이블에 실시간 저장.
4. **리포트 분석**: 누적된 대화 데이터를 분석하여 '진로 키워드' 추출.
